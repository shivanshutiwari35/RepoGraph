# Evaluation Specification — RepoScope

Engineering plans generated by LLMs must be measured to ensure reliability. RepoScope uses a programmatic and LLM-assisted Evaluation Harness to score and validate plan quality.

---

## 1. Metric Framework

| Metric | Type | Description |
| :--- | :--- | :--- |
| **Dependency Coverage** | Programmatic | Checks if all files identified as downstream dependencies in the NetworkX graph are mentioned in the plan's "Affected Components". |
| **Risk Detection** | Hybrid | Evaluates if the plan outlines risks for highly-connected graph nodes (e.g. database connections, auth middlewares). |
| **Rollback Included** | Programmatic (Regex) | Confirms the plan contains a rollback section with recovery steps. |
| **Clarification Score** | Deterministic | Checks if needed clarifications were gathered before the plan was generated. |
| **Confidence Accuracy** | LLM-graded | Assesses if the planner's confidence score aligns with the number of unknowns. |

---

## 2. Plan Scopes & Score Calculation

The final score is out of `100`, calculated as:
- **Dependency Coverage (30 points)**: `(Number of correctly identified affected components / Total topological dependencies) * 30`
- **Risk Mitigation (20 points)**: Verification that key dependencies have risk mitigations outlined.
- **Rollback Feasibility (20 points)**: Presence of explicit rollback steps.
- **Ambiguity Clarification (20 points)**: Score penalized if router identified clarification needs but none were collected.
- **Logical Flow (10 points)**: Planner flow check.
